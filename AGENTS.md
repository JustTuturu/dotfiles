# Tuturu's Dotfiles — Agent Guide

## Repo Layout

This is a **Fedora-based Hyprland** dotfiles repo managed with **GNU Stow**.
Packages are at the repo root (e.g. `hypr/`, `zsh/`, `ghostty/`), **not** inside a `.config/` folder.

## How to Add a New Config Package

1. Create the directory structure: `newpkg/.config/newpkg/config.file`
2. Add `newpkg` to `packages/stow.txt`
3. Stow it: `stow -t ~ newpkg`

## Theming Rules

- **All color theming must go through matugen** where the target app supports it.
- Matugen templates live in `matugen/.config/matugen/templates/*.mustache`.
- Generated output goes to `~/.config/matugen/generated/` (or app-specific paths like `~/.config/eza/theme.yml`).
- **Never commit generated files.** They are runtime artifacts.
- Static color files are only acceptable for apps that cannot consume dynamic colors (e.g. Zed editor themes, Tmux plugin themes, compiled Bat themes).

## Matugen Template Conventions

- Use `{{colors.primary.default.hex}}` for accent colors.
- Use `{{colors.surface_container_lowest.default.hex}}` for backgrounds.
- Use `{{colors.on_surface.default.hex}}` for foregrounds.
- Use `{{colors.error.default.hex}}` for destructive/error states.
- For RGBA (Hyprland), use `{{colors.primary.default.hex_alpha_stripped}}` with your own alpha.
- Add new template sections to `matugen/.config/matugen/config.toml`.

## Hand-Edited vs Generated Files

| File | Status |
|------|--------|
| `matugen/templates/*.mustache` | Hand-edited |
| `matugen/generated/*` | Generated — **never edit** |
| `noctalia/settings.json` | GUI-exported from Noctalia settings panel |
| `noctalia/plugins.json` | GUI-exported |
| `gtk-*/noctalia.css` | Generated by Noctalia template processor |
| `hypr/conf/*.conf` | Hand-edited |
| `install.sh` | Hand-edited |

## Zsh Conventions

- `.zshrc` sources `~/.config/matugen/generated/zsh-highlight.zsh` **before** `zsh-syntax-highlighting` plugin.
- Deferred plugins use `zinit ice wait'0' lucid`.
- NVM is lazy-loaded via a wrapper function.
- Do not duplicate FZF initialization (`eval "$(fzf --zsh)"` is sufficient; do not also source `~/.fzf.zsh`).

## Stow Conventions

- `install.sh` passes `--no-folding` to stow so directories are not symlinked as a whole.
- The `find . -type d + mkdir -p` pre-creation dance has been removed in favor of `--no-folding`.
- If a package needs a runtime-generated file in the same directory (e.g. `eza/theme.yml`), matugen overwrites the stow-created symlink. This is intentional. After a `stow -R`, rerun matugen.

## Known Gotchas

- `hyprlock.conf` uses `source = ~/.config/matugen/generated/hyprlock-colors.conf` (supported by hyprlock's Hyprland config backend).
- Ghostty uses both a static `theme = noctalia` fallback **and** a dynamic `config-file` include. The include overrides palette colors when matugen has run.
- Starship config is fully generated by matugen to `~/.config/starship.toml`; there is no static starship.toml in the repo.
- `9router` autostart appears once in `hypr/conf/autostart.conf`.
- Wallpaper path in `install.sh` intentionally references `~/Pictures/Wallpapers/` (user-managed), not the repo `wallpapers/` directory.
