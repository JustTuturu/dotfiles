## Overview

Fedora-based Hyprland dotfiles managed with **GNU Stow**. All color theming is driven by [**matugen**](https://github.com/InioX/matugen) (Material You generator) from your wallpaper.

| Component | Choice |
|-----------|--------|
| OS | Fedora Linux |
| WM | Hyprland + UWSM |
| Shell | Zsh (Zinit) + Starship |
| Bar / Launcher | Noctalia (Quickshell) |
| Terminal | Ghostty |
| Editor | Zed |
| File Manager | Yazi |
| Multiplexer | Tmux |
| Theming | Matugen → dynamic colors for Hyprland, Ghostty, Eza, FZF, Zsh, Rofi, Starship, Hyprlock |

## Directory Structure

```
~/dotfiles/
├── hypr/            → ~/.config/hypr
├── zsh/             → ~/.zshrc
├── ghostty/         → ~/.config/ghostty
├── matugen/         → ~/.config/matugen
├── eza/             → ~/.config/eza
├── yazi/            → ~/.config/yazi
├── starship/        → ~/.config/starship.toml (generated by matugen)
├── tmux/            → ~/.config/tmux
├── zed/             → ~/.config/zed
├── noctalia/        → ~/.config/noctalia
├── fastfetch/       → ~/.config/fastfetch
├── fontconfig/      → ~/.config/fontconfig
├── gtk/             → ~/.config/gtk-3.0 & gtk-4.0
├── fcitx5/          → ~/.config/fcitx5
├── bat/             → ~/.config/bat
├── lazygit/         → ~/.config/lazygit
├── uv/              → ~/.config/uv
├── wallpapers/      → (not stowed, referenced by matugen)
├── packages/        → package lists for install.sh
└── install.sh       → Fedora bootstrap installer
```

## Quick Start

```bash
cd ~/dotfiles

# Install everything (Fedora only)
./install.sh install

# Or stow configs manually
stow -t ~ $(cat packages/stow.txt)

# Generate colors from wallpaper
matugen image ~/Pictures/Wallpapers/wallpaper.jpg --prefer darkness
```

## Stow Cheatsheet

```bash
cd ~/dotfiles

# Stow all
stow -t ~ $(cat packages/stow.txt)

# Stow one package
stow -t ~ hypr

# Unstow
stow -t ~ -D hypr

# Restow
stow -t ~ -R hypr
```

## Theming Architecture

**Matugen** reads your wallpaper and generates a Material You color palette. These colors flow into every supported app via mustache templates:

| Template | Output | Consumer |
|----------|--------|----------|
| `hyprland.conf.mustache` | `~/.config/matugen/generated/hyprland-colors.conf` | Hyprland (`source = ...`) |
| `ghostty.conf.mustache` | `~/.config/matugen/generated/ghostty-colors.conf` | Ghostty (`config-file = ...`) |
| `eza.yml.mustache` | `~/.config/eza/theme.yml` | Eza |
| `zsh-highlight.zsh.mustache` | `~/.config/matugen/generated/zsh-highlight.zsh` | Zsh (sourced) |
| `zsh-fzf-colors.zsh.mustache` | `~/.config/matugen/generated/zsh-fzf-colors.zsh` | Zsh (sourced) |
| `starship.toml.mustache` | `~/.config/starship.toml` | Starship |
| `hyprlock.conf.mustache` | `~/.config/matugen/generated/hyprlock-colors.conf` | Hyprlock (`source = ...`) |
| `rofi.rasi.mustache` | `~/.config/matugen/generated/rofi-colors.rasi` | Rofi |
| `quickshell-colors.json.mustache` | `/tmp/qs_colors.json` | Noctalia / Quickshell |

> **Note:** Some tools (Tmux, Zed, Bat) do not support dynamic Material You theming and use static Catppuccin/Macchiato fallbacks.

```bash
ls -la ~/.config/ | grep "^l"

```

## Notes

- Always `cd ~/dotfiles` before running stow.
- Matugen-generated files are **not** committed; they are created at runtime.
- If stow conflicts exist, back up the existing file before stowing.
- The `install.sh` uses `--no-folding` to ensure file-level symlinks.
- `quickshell` and `rofi` symlinks in `~/.config` may point outside this repo (managed by Noctalia).
